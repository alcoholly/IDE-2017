Positive and Negative Aspects about the 2 Visualisations

First visualisation:

This visualisation use an interactive graph to present themes of Polish books (nodes) and how they are related to each other, connected via links that indicate a frequent co-occurence between the themes.

Positive

It is easy to estimate the magnitude of co-occurences for each theme by looking at the sizes of the nodes, with an accompanying legend on the side showing what minimum (200) and maximum (2000) occurrences look like. This element also allows us to easily distinguish the hubs of the graph (Love, Woman, Death). The legend on the side that shows the correspondence of node colors to literature genres, adds an extra level of information to the graph in a compact way. While at first it’s a bit difficult to see how the themes are actually related, with lots of links going everywhere, when hovering over one node its links and its neighbors get highlighted, making it easy to see the connections of a theme you are interested in.

Negative

As mentioned before, there are many links in the graph, and it looks a bit cluttered. The edges between the themes do not actually add any value to the visualization (in its current form) when the nodes aren’t hovered; everything would look clearer and cleaner if the links only appeared when the user hovers over a node. Moreover, each time the page is loaded, the graph is randomly positioned, and sometimes it overlaps with the legend to the right or ends up outside the box where it is presented, making it difficult or impossible to read some of the text. There are also areas in the graph where too many nodes are positioned in the same space, which also causes the node labels to overlap. Lastly, when the color of a node is mixed (i.e. when the theme appears across different genres of literature) it is not always easy to estimate the proportion of the basic colors it consists of and thus, it is difficult to understand how some of the presented themes are distributed over different genres of literature.

Second visualisation:

This visualisation aims to present how famous creative people structure their day based on 6 categories of daily routines- namely SLEEP, CREATIVE WORK, DAY JOB/ADMIN, FOOD/LEISURE, EXERCISE and OTHER. For each individual a 24h timeline is structured into consecutively aligned colored bars representing the above mentioned activities.

Positive

The visualisation allows for accessing multiple levels of details ( e.g. by hovering over each of the bars we can get more specific information the daily routines of an individual) The design also supports filtering the activity categories so we can both have a broad overview of all the categorical data as well as access and compare selected groups or individual categories separately. Lastly, the use of the color is effective since prominent colors are used to highlight the categories that are most likely to be of interest to the user (e.g. “creative work” category is represented using red) while a more neutral color (gray) is used for the less well-defined category “other”.


Negative

The data for each category are fragmented over time and there is no option offered to aggregate them per category for each individual. Thus, it is difficult to form a clear idea about the total amount of time each individual spends on a specific activity and compare the differences between different individuals (see image above). Another negative aspect is that it is not easy to compare the daily routines of people that are not positioned close to each other in the row index. For example while we can easily compare the daily routines between Haruki Murakami and Voltaire or Benjamin Franklin, it is quite difficult to compare the routines of Honore de Balzac and Pablo Picasso.

Individual part: Describe an interactive visualization that you would like to be able to build after this course has finished

Ioanna Arkoudi:

This semester I have taken Interactive Graphs & Interactions as an elective course in DTU and I am currently working on a project that aims to explore the musical preferences of people around the world using data from Last.fm (more information about this project are available here). After Interactive Data Exploration finishes, I would like to be able to use the collected data and create an interactive graph with the set of the most popular artists around the world as nodes. The edges between the nodes are added by thresholding a similarity score between the artists - that is also provided by Last.fm. A basic description of the interactive visualisation I would like to implement is provided below:

  1. The visualisation should include the graph (nodes and edges) with the sizes of the nodes corresponding to their degree - so that the user gets an idea about the significance of each node in the graph and is able to easily spot the hubs. Next to the graph, there should be a section that contains basic stats about the graph (number of nodes, number of edges, average degree etc).
  2. When the user clicks on a node, then its connections to other nodes are highlighted. The content of the section that appears next to the graph also changes to include basic information about the selected node (degree, betweenness centrality etc).
  3. I would additionally like to incorporate an on-off "community mode". This mode should be off by default but when the user activates it, then the nodes of each community are colorized differently to indicate the partition of the network. The section next to the graph now should contain basic information about the community partition of the graph (Modularity score, number of communities, number of nodes in each community etc).
  4. Lastly, when the user clicks on a node of a community then this community is highlighted and the informative section next to the graph changes again to contain information about the selected community (number of nodes, number of inside links, number of outside links to other communities).
